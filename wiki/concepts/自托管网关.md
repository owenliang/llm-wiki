# 自托管网关

自托管网关是指运行在自己硬件上的服务网关，用户完全控制数据和运行环境，无需依赖第三方托管服务。

## 核心特征

- **数据自主**: 数据存储在本地，不依赖云服务
- **隐私保护**: 敏感信息不出本地
- **完全控制**: 自定义配置、扩展和集成
- **无订阅费**: 只需支付自己的硬件和 API 费用

## 典型应用场景

### AI Agent 网关

如 [[OpenClaw]]：
- 连接多个聊天渠道到 AI Agent
- 本地处理消息路由和会话管理
- 自主选择 LLM 提供商

### 优势对比

| 特性 | 自托管 | 托管服务 |
|------|--------|----------|
| 数据隐私 | 完全控制 | 依赖服务商 |
| 定制化 | 完全自由 | 受限 |
| 成本 | 硬件 + API | 订阅费 |
| 维护责任 | 用户承担 | 服务商承担 |
| 可用性 | 依赖本地网络 | 服务商 SLA |

## 技术栈

- **运行时**: Node.js、Python、Docker
- **配置**: JSON/YAML 配置文件
- **访问**: 本地 Web UI、SSH、Tailscale
